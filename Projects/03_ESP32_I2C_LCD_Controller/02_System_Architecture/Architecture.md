# System Architecture

## System Overview

The system is designed around an ESP32-S3 microcontroller acting as the central processing unit responsible for display management, keypad scanning, and user interface control.

A 128×64 graphical LCD is connected through an MCP23017 I²C GPIO expander, allowing the display to be controlled while significantly reducing the number of GPIO pins required from the ESP32.

The architecture was developed with the following objectives:

* Minimize GPIO consumption.
* Reduce PCB routing complexity.
* Maintain a compact PCB layout.
* Enable future firmware scalability.
* Support graphical user interface development.

---

## High-Level Functional Flow

1. User input is received through an external keypad.
2. ESP32-S3 processes user commands.
3. Display data is generated by firmware.
4. LCD commands are transmitted through the I²C bus.
5. MCP23017 expands the I²C commands into parallel control signals required by the graphical LCD.
6. The graphical LCD updates the displayed information.

---

## ESP32-S3 Selection

The ESP32-S3 was selected as the primary controller because it provides:

* High processing capability.
* Large Flash and RAM resources.
* Multiple communication interfaces.
* Low-cost development ecosystem.
* Strong open-source software support.

The controller provides sufficient resources to manage graphical display rendering while simultaneously handling keypad inputs and future communication features.

---

## Graphical LCD Selection

A 128×64 graphical LCD was selected to provide richer visualization compared to standard character LCD modules.

Advantages include:

* Bitmap graphics support.
* Custom icons and symbols.
* Menu-based interfaces.
* Flexible user interface design.

This makes the display suitable for industrial and embedded human-machine interface applications.

---

## Why an I²C GPIO Expander Was Used

Graphical LCD modules typically require a large number of control and data lines.

Direct LCD interfacing would consume a significant portion of the ESP32 GPIO resources.

To solve this problem, an MCP23017 I²C GPIO expander was introduced.

Benefits include:

* Significant GPIO reduction.
* Simplified PCB routing.
* Easier firmware scalability.
* Additional GPIO availability for future peripherals.

Only the SDA and SCL lines are required from the ESP32 to communicate with the expander.

---

## MCP23017 Selection

The MCP23017 was selected because it provides:

* 16 programmable GPIO pins.
* I²C communication interface.
* Wide operating voltage range.
* Excellent software support.
* Industrial-proven reliability.

The device acts as a bridge between the serial I²C bus and the parallel interface required by the graphical LCD.

---

## Display Firmware Architecture

The display firmware was designed around commonly used graphical display libraries.

The design was evaluated with:

* openGLCD Library
* U8g2 Library

These libraries provide:

* Character rendering
* Bitmap drawing
* Menu generation
* Font management
* Display abstraction

The software layer allows application code to remain independent from the low-level display hardware implementation.

---

## Keypad Interface Architecture

An external keypad interface is provided for user interaction.

The keypad enables:

* Menu navigation
* User selections
* System configuration
* Human-machine interaction

The keypad interface remains independent of the display interface, improving software modularity.

---

## Power Architecture

The system receives power from an external DC source.

Power is distributed to:

* ESP32-S3
* MCP23017
* Graphical LCD
* Keypad interface circuitry

Decoupling capacitors are placed close to each integrated circuit to improve supply stability and reduce switching noise.

---

## Design Philosophy

The architecture prioritizes:

* GPIO efficiency
* Modular firmware development
* Compact PCB implementation
* Manufacturability
* Future scalability

The resulting design provides a flexible embedded display platform that can be reused in future HMI-based products.
